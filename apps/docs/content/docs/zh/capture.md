# 捕获流量

**Capture** 工作区会实时显示经过 PostGate 的流量。

## 启动与暂停

点击 **Start** 开始监听代理端口。**Pause** 只会暂停向列表添加新记录，代理仍会继续转发流量，也不会删除已经捕获的请求。按钮旁的地址可以直接复制到浏览器或系统代理设置中。

点击垃圾桶按钮可以清空捕获历史。对于长时间运行的会话，可以在 **Settings** 中调整数据保留和持久化设置。

## 筛选请求

搜索框会匹配 URL、主机名和路径。筛选菜单还支持：

- HTTP 方法
- 状态码范围
- 内容类型
- 主机名
- 协议
- 是否匹配规则

筛选只影响可见列表，不会阻止代理转发流量。

## 查看请求

选择一行后即可查看请求和响应详情。较大的正文会按需加载；文本、JSON、HTML、图片和二进制数据会使用对应的预览方式。耗时瀑布图会分别展示建立连接、发送请求、等待上游和接收响应等阶段。

分享捕获数据前，请将 `Authorization`、`Cookie` 等敏感请求头视为凭证，并先移除或脱敏。

## 导出与 Replay

捕获结果可以导出为 HAR，供浏览器 DevTools 或其他 HTTP 工具使用。请求也可以导入 [Replay](/docs/replay)，在修改 URL、方法、查询参数、请求头和正文后重复执行。

## 流式流量

WebSocket 和 Server-Sent Events（SSE）连接会持续显示后续消息。HTTP/2 与 HTTP/1.1 共用同一套捕获和规则处理链路。只有启用了 QUIC 功能的构建才提供实验性 HTTP/3 监听入口。
