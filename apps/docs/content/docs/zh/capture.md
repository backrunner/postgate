# 捕获流量

Capture 工作区用于查看所有通过 PostGate 的实时流量。

## 启动与暂停

点击 **Start** 开始监听代理端口。**Pause** 只会停止监听，不会删除已经捕获的请求。按钮旁的地址可以直接复制到浏览器或系统代理设置中。

垃圾桶按钮会清空已保存的捕获历史。是否持久化捕获数据可以单独设置，因此长时间运行时无需把所有记录都保留在内存中。

## 筛选请求

搜索框会匹配 URL、主机名和路径。筛选菜单还支持：

- HTTP 方法
- 状态码范围
- Content Type
- 主机名
- 协议
- 是否匹配规则

筛选只影响可见列表，不会阻止代理转发流量。

## 查看请求

选择一行后可以查看请求和响应详情。较大的正文会按需加载；文本、JSON、HTML、图片和二进制数据也会使用各自合适的预览方式。耗时瀑布图会分别展示连接、发送请求、等待上游和接收响应等阶段。

分享捕获数据前，请将 `Authorization`、`Cookie` 等敏感请求头按凭证处理。

## 导出与 Replay

捕获结果可以导出为 HAR，供浏览器 DevTools 或其他 HTTP 工具使用。请求也可以导入 [Replay](/docs/replay)，在修改 URL、方法、查询参数、请求头和正文后重复执行。

## 流式流量

WebSocket 和 Server-Sent Events 连接会持续显示后续消息。HTTP/2 与 HTTP/1.1 共用同一套捕获和规则处理链路。只有启用了 QUIC 功能的构建才提供实验性 HTTP/3 入口。
