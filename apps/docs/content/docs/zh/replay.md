# 重放请求

Replay 可以把一次捕获的请求转成可重复执行的测试。保存的请求会归入集合，并保留执行历史。

## 从 Capture 导入

在 Capture 中选择请求并导入 Replay。PostGate 会复制请求方法、URL、查询参数、请求头和正文，所有内容都可以继续编辑。

保存或分享请求前，请先检查其中的凭证和会话 Cookie。Replay 会原样发送编辑器当前显示的值。

## 新建请求

也可以从零开始：

1. 选择或创建集合。
2. 输入方法和完整 URL。
3. 添加查询参数和请求头。
4. 选择正文类型，输入文本、JSON、表单或 multipart 数据。
5. 执行请求。

响应区会显示状态码、响应头、正文和耗时。修改规则或本地服务后，可以通过执行历史对比前后的结果。

## 管理集合

集合支持创建、重命名和删除，请求也可以复制或移动到其他集合。启用对应选项后，配置文件导出和同步会包含 Replay 集合及其中的请求。

## Replay 与规则

Replay 会从桌面应用直接发起请求，不会自动经过 PostGate 代理，因此代理规则不会应用到 Replay 请求。要验证代理规则，请使用已配置 PostGate 代理的浏览器或其他客户端发起请求，再到 **Capture** 中查看对应记录。
