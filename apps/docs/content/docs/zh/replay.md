# 重放请求

Replay 可以把一次捕获转成可重复执行的本地测试。请求保存在集合中，并保留执行历史。

## 从 Capture 导入

在 Capture 中选择请求并导入 Replay。PostGate 会复制请求方法、URL、查询参数、请求头和正文，所有内容都可以继续编辑。

保存或分享前检查凭证与会话 Cookie。Replay 会发送编辑器当前显示的值。

## 新建请求

也可以从零开始：

1. 选择或创建集合。
2. 输入方法和完整 URL。
3. 添加查询参数和请求头。
4. 选择正文类型，输入文本、JSON、表单或 multipart 数据。
5. 执行请求。

响应区会显示状态码、响应头、正文和耗时。修改规则或本地服务后，可以通过执行历史对比前后的结果。

## 管理集合

集合支持创建、重命名和删除，请求也可以复制或移动到其他集合。启用对应选项后，配置档案导出和同步会包含 Replay 集合及其中的请求。

## Replay 与规则

Replay 会从桌面应用直接发起保存的请求。验证代理规则时，请确认请求确实经过当前 PostGate 代理链路，并与 Capture 中的对应记录进行比较。
